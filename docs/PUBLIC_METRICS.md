## Beachhead

The first vertical should be public-sector or regulated procurement evidence, because that workflow has an acute need for inspectable AI output records and a natural buyer lever: contract clauses.

First beachhead outcome:

> one buyer or evaluator requires a signed TrustEnvelope, or functionally equivalent open evidence object, for a bounded AI workflow.

## Production Evidence Definition

Do not call a deployment "undeniable production deployment" unless all of these are true:

- real organization;
- real workflow;
- real signing key controlled by that organization or its operator;
- DNS-hosted manifest;
- at least one signed output;
- independent verifier result;
- permission to publish a redacted proof note.

## Dashboard Design

The first public dashboard can be static. It should show:

- current counter;
- target threshold;
- last updated date;
- link to evidence;
- "not yet" state when the counter is zero.

No hidden telemetry is allowed. Any public verifier-run counter must be opt-in and documented.
